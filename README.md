# 🎓 SEAS: Smart-Edu Agentic System
### 基于 Agent 架构的“诊断-检索-干预”自动化教学中枢

SEAS 是一个集成型 AI 框架，旨在解决教育场景中“数据孤岛”与“通用模型缺乏学科深度”的痛点。系统通过 **MCP 协议** 获取实时成绩，利用 **微调 RAG** 提供专家级解析，并由 **自主 Agent** 编排教学干预工作流。

---

## 📖 项目做了什么？ (Core Functionality)

本系统实现了从**原始数据**到**个性化教学行动**的全流程自动化：

### 1. 交互式多维数据分析 (via MCP & LUI)
* **意图驱动查询**：用户（老师/家长）通过自然语言询问“班级哪部分知识点掌握最薄弱？”，系统自动将指令转化为精准的 SQL 逻辑。
* **工具化数据操作**：通过 **Model Context Protocol (MCP)** 封装底层教务 API，实现模型对学生分数的安全、受控访问，避免了直接操作数据库带来的风险。

### 2. 学科深度知识检索 (via Fine-tuned RAG)
* **精准错题匹配**：当系统定位到学生弱项（如：牛顿第二定律）时，RAG 模块会从海量题库中检索对应的深度解析。
* **领域微调支撑**：通过对 Embedding 模型进行**学科垂直微调 (SFT)**，确保 AI 生成的解题思路符合教学大纲要求，而非通用模型的模糊回答。

### 3. 闭环教学干预工作流 (via Autonomous Agent)
* **自动诊断与规划**：Agent 监控到成绩异常波动后，自主启动“诊断-搜寻-推送”任务流。
* **个性化策略生成**：根据学生过往表现，动态决定干预强度（如：仅发送解析、推荐类似题、或标记为需人工介入）。

## 🏗️ 技术架构 (System Architecture)

系统采用**分层解耦**的设计模式，确保各模块可独立扩展：

* **感知执行层 (MCP Server)**: 使用 Python FastAPI 封装，定义强类型的 JSON Schema 工具集，负责与教务数据库（PostgreSQL）通信。
* **知识大脑 (RAG Engine)**: 基于 ChromaDB 向量数据库，引入 **Rerank (重排序)** 机制，确保高价值教学资料的召回率。
* **编排中心 (Agent Controller)**: 基于 **LangGraph** 构建，利用状态机（State Machine）管理复杂的任务流和错误重试逻辑。

---

## 🛠️ 工程亮点 (Technical Highlights)

* **参数幻觉抑制**: 利用 MCP 协议的约束性，强制模型按预定义格式传参，显著降低了误报率。
* **混合检索优化**: 在 RAG 模块中结合了语义搜索与关键词搜索，精准处理数理化公式等硬核内容。
* **异步干预逻辑**: 采用异步任务处理 Agent 的执行流，支持大规模班级的并发诊断与个性化推送。
